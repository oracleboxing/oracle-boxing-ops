## Scope

Provide technical development and integration services on a project basis. Build automations, fix technical issues, implement integrations, and maintain technical infrastructure. This is not a full-time role; work is engaged via Statement of Work (SOW) as needed.

---

## Engagement Model

### How Projects Are Engaged

1. **Backlog Review:** Operations Manager maintains a prioritised backlog of technical work
2. **SOW Creation:** For each project, create a Statement of Work including:
   - Project description and requirements
   - Deliverables and acceptance criteria
   - Timeline and milestones
   - Rate/budget
   - Communication expectations
3. **SOW Agreement:** Both parties agree to terms before work begins
4. **Execution:** Contractor completes work per SOW
5. **Review & Acceptance:** Operations Manager reviews and accepts per SOW criteria
6. **Payment:** Contractor paid per agreed terms (typically monthly with other contractors)

### SOW Template Elements

| Element | Description |
| --- | --- |
| **Project Title** | Clear name for the work |
| **Background** | Context and why this work is needed |
| **Requirements** | What needs to be built or fixed |
| **Deliverables** | Specific outputs expected |
| **Acceptance Criteria** | How we'll know it's done correctly |
| **Timeline** | Expected start and end dates |
| **Rate/Budget** | Agreed compensation |
| **Communication** | Check-in cadence, tools, escalation |

## Types of Work

### Automations & Integrations
- [ ] Build new Make.com workflows
- [ ] Modify existing automations
- [ ] Integrate third-party tools (Stripe, Skool, etc.)
- [ ] Set up webhooks and data flows

### Technical Fixes
- [ ] Debug and fix software issues
- [ ] Resolve integration failures
- [ ] Fix website functionality problems
- [ ] Address payment processing issues

### Technical Improvements
- [ ] Implement new features on website/portal
- [ ] Improve existing systems performance
- [ ] Build internal tools or dashboards
- [ ] Set up monitoring and alerting

### Documentation & Handoff
- [ ] Document technical systems and configurations
- [ ] Create runbooks for common issues
- [ ] Maintain change log for all modifications
- [ ] Ensure knowledge transfer for ongoing maintenance

---

## Contractor Requirements

### Skills Expected
- Relevant technical skills for the project type
- Experience with key tools (Make.com, Stripe, web technologies)
- Clear communication and documentation habits
- Ability to work independently within SOW scope

### Communication Standards
- Acknowledge SOW within 24 hours
- Provide progress updates per agreed cadence
- Raise blockers or scope questions promptly
- Deliver work by agreed timeline or communicate delays early

### Deliverable Standards
- Meet acceptance criteria defined in SOW
- Provide documentation for all changes
- Update change log with modifications
- Test changes before submitting for review

---

## Decision Rights

### Contractor Decides
- Technical implementation approach within requirements
- Tool and library choices within constraints
- Work schedule (as long as timeline is met)

### Operations Manager Decides
- Project prioritisation and backlog order
- SOW terms and acceptance criteria
- Whether to accept completed work
- Budget allocation for technical work

### Must Escalate (to Managing Director)
- Budget increases beyond approved amount
- Timeline extensions affecting other departments
- Technical decisions with significant business impact

---

## KPIs / Success Measures

- **Delivery:** Work delivered per SOW timeline
- **Quality:** Work accepted without major revisions
- **Documentation:** All changes documented and logged
- **Communication:** Updates provided per agreed cadence
- **Budget:** Work completed within agreed budget

## Current Backlog Categories

The Operations Manager maintains a backlog of technical work, typically including:

- **Automation improvements:** Make.com workflow enhancements
- **Integration fixes:** Stripe, Skool, email tool issues
- **Website updates:** Functionality improvements, bug fixes
- **New features:** Dashboard, tools, or capability additions
- **Technical debt:** Clean-up, documentation, monitoring
